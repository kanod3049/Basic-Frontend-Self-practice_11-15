Week 11: การจัดการ DOM และ Event Handling
เรียนรู้การใช้งาน Event Listener รูปแบบต่างๆ click, submit, input เพื่อสร้าง Interact UI
ฝึกฝนการอัปเดตข้อมูลบนหน้าเว็บแบบ Real-time โดยไม่ต้องรีโหลดหน้าจอ
ประยุกต์ใช้เทคนิค Event Delegation เพื่อลดจำนวน Listener ช่วยเพิ่มประสิทธิภาพการทำงานของเว็บ
เชื่อมต่อข้อมูลระหว่าง DOM กับ localStorage เพื่อบันทึกสถานะข้อมูลให้คงอยู่ต่อเนื่อง

Week 12: การเชื่อมต่อข้อมูลด้วย Fetch API และ Async/Await
เรียนรู้การใช้ Fetch API เพื่อดึงข้อมูลจากแหล่งภายนอก มาแสดงผล
ปรับรูปแบบการเขียนโค้ดเป็น Async/Await เพื่อจัดการการทำงานแบบ Asynchronous ให้เป็นลำดับขั้นตอนและอ่านง่ายขึ้น
ทบทวนและนำ DOM Methods มาใช้ร่วมกับข้อมูลที่ได้จาก API เพื่อทำการ เพิ่ม, ลบ, และแก้ไขข้อมูลหน้าเว็บ

Week 13: Modular Architecture และการปรับปรุง UI
เริ่มจัดระเบียบโค้ดโดยแยกไฟล์เป็น ES Modules import/export เพื่อให้ดูแลรักษาได้ง่าย
พัฒนา UI Feature เพิ่มเติม เช่น ระบบค้นหา และการเรียงลำดับข้อมูล 
เพิ่มระบบป้องกันความผิดพลาดด้วย Confirmation Dialog ก่อนทำการลบข้อมูลสำคัญ

Week 14: Code Refactoring
ทำการ Refactor โครงสร้าง Frontend ใหม่ตามคำแนะนำของอาจารย์ โดยแยกส่วนการเชื่อมต่อข้อมูล API Handling ออกจากส่วน Business Logic
สร้าง Dialog Module แยกออกมาเป็นไฟล์เฉพาะ เพื่อจัดการ Modal/Popup ให้เป็นมาตรฐานเดียวกันทั้งโปรเจกต์ ลดความซ้ำซ้อนของโค้ด และทำให้การจัดการ UI State มีระเบียบมากขึ้น

Week 15: Mock API & Complex Logic
นำหลักการ Asynchronous มาประยุกต์ใช้ร่วมกับ mock API เพื่อจำลองการตอบกลับจาก Server ทำให้สามารถพัฒนาฟีเจอร์จองรายวิชา และทดสอบ UI ได้ทันทีโดยไม่ต้องรอ Backend
พัฒนาระบบคำนวณหน่วยกิต และการแสดงผลข้อมูลรายวิชาแบบ Dynamic
ระบบด้วยการตรวจสอบช่วงเวลาจอง โดยมีการปิดกั้นปุ่มกดที่หน้า UI และเขียนดักจับ Error เพื่อรองรับกรณีที่ระบบปิดรับการจอง
